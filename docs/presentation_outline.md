# Presentation Outline

1. **Problem & Goal**
   - Make a patient tutor that remembers context and can reference a small knowledge base.

2. **Approach**
   - Basic LLM use (Week 2), memory (Week 3), RAG (Week 4), Conversational RAG (Week 5), Web UI (Week 6).

3. **Architecture**
   - Components and data flow (see docs/architecture.md).

4. **Demo**
   - Show CLI and web interface.

5. **Results & Challenges**
   - Quality vs. small models; prompt sensitivity; managing hallucinations.

6. **Future Work**
   - Better models, tool use, evaluation metrics, persistence of user profiles.
